Investiga en diferentes sitios web los siguientes tópicos y describe cada uno de ellos:

Tema 1: Elementos de Páginas Web

  1. Orígenes de la Web
    1. Web 2.0

      2. Interactividad: Los usuarios interactúan activamente con el contenido. Contenido generado por el usuario: Plataformas como redes sociales y wikis. Colaboración: Herramientas como Google Docs. API y mashups: Integración de servicios y datos.

    2. Web 3.0 3

      3. Semántica: Mejora en la comprensión de datos por máquinas. Descentralización: Uso de blockchain para mayor seguridad y privacidad. Inteligencia artificial: Personalización de experiencias. Conectividad: Integración fluida de aplicaciones y servicios.

    3. La W3C y los estándares

      4. Objetivos: Fomentar interoperabilidad y accesibilidad. Normas: Desarrollo de estándares como HTML y CSS. Accesibilidad: Garantiza que la web sea usable para todos. Iniciativas: Aborda privacidad, seguridad e inclusión digital.

  2. Lenguajes de etiquetas
    1. SGML

      2. Descripción: Lenguaje de marcado estándar para definir documentos estructurados. Uso: Base para la creación de otros lenguajes de marcado, incluyendo HTML y XML. Características: Flexible y extensible, pero complejo de implementar.

    2. XML

      3. Descripción: Lenguaje de marcado que permite definir estructuras de datos personalizadas. Uso: Intercambio de información entre sistemas y almacenamiento de datos. Características: Legible por humanos y máquinas; permite crear etiquetas personalizadas.

    3. HTML

      4. Descripción: Lenguaje de marcado utilizado para crear y estructurar páginas web. Uso: Define elementos como texto, imágenes, enlaces y formularios. Características: Estructura básica de la web; fácil de aprender.

    4. XHTML

      5. Descripción: Versión de HTML que sigue las reglas de XML. Uso: Proporciona una estructura más rigurosa y estándar para páginas web. Características: Mejora la interoperabilidad; requiere un marcado bien formado.

    5. HTML5

      6. Descripción: Última versión de HTML, incorpora nuevas características y elementos. Uso: Soporta multimedia (audio, video), gráficos (canvas) y aplicaciones web. Características: Enfocado en la semántica, accesibilidad y mayor soporte para dispositivos móviles.

  3. Otros lenguajes relacionados al desarrollo web
    1. CSS/CSS3

      2. Descripción: Lenguaje de estilo para definir la presentación de documentos HTML. Uso: Controla el diseño, colores y fuentes. Características: CSS3 incluye flexbox, grid, animaciones y media queries para diseños responsivos.

    2. JavaScript

      3. Descripción: Lenguaje de programación para agregar interactividad a las páginas web. Uso: Manipula el DOM y gestiona eventos del usuario. Características: Soporta programación orientada a objetos, asincronía y se utiliza con bibliotecas como React y Angular.

    3. Ajax

      4. Descripción: Técnica para actualizar contenido de la página sin recargarla. Uso: Carga de datos en segundo plano, ideal para formularios y contenido dinámico. Características: Utiliza XMLHttpRequest o Fetch API y maneja datos en JSON o XML.

    4. Tecnologías y lenguajes del lado del servidor

      5. Descripción: Lenguajes que se ejecutan en el servidor para generar contenido dinámico. Ejemplos: PHP, Python (Django, Flask), Ruby (Rails), Java (Spring), Node.js. Uso: Procesan solicitudes del cliente y gestionan bases de datos.

    5. Navegadores

      6. Descripción: Aplicaciones para acceder y visualizar contenido web. Ejemplos: Google Chrome, Mozilla Firefox, Safari, Microsoft Edge. Características: Interpretan HTML, CSS y JavaScript; cada uno tiene herramientas de desarrollo integradas.

    6. Motores renderizados

      7. Descripción: Componentes que interpretan el código HTML, CSS y JavaScript en los navegadores. Ejemplos: Blink (Chrome), WebKit (Safari), Gecko (Firefox). Características: Varia en rendimiento y compatibilidad, influyendo en la experiencia del usuario.

  4. Herramientas de desarrollo
    1. Software WYSIWYG

      2. Descripción: Programas que permiten diseñar páginas web visualmente, mostrando cómo se verán en el navegador. Ejemplos: Adobe Dreamweaver, Wix, WordPress (editor visual). Características: Permiten arrastrar y soltar elementos, facilitando el diseño sin necesidad de escribir código.

    2. Otras alternativas
      1. Figma

        2. Descripción: Herramienta de diseño de interfaces y prototipado colaborativo. Uso: Permite crear diseños de UI/UX y prototipos interactivos. Características: Colaboración en tiempo real, diseño basado en vectores y exportación fácil a diferentes formatos.

      2. InVisión

        3. Descripción: Plataforma para prototipado y colaboración en diseño. Uso: Facilita la creación de prototipos interactivos y la retroalimentación del equipo. Características: Herramientas de comentarios y revisiones, integración con otras herramientas de diseño.

      3. Zeplin

        4. Descripción: Herramienta que ayuda a la transición entre diseño y desarrollo. Uso: Proporciona especificaciones de diseño y activos para desarrolladores a partir de archivos de diseño. Características: Genera estilos CSS, exporta activos y facilita la comunicación entre diseñadores y desarrolladores.

      4. GitHub

        5. Descripción: Plataforma de gestión de versiones y colaboración en proyectos de software. Uso: Permite almacenar, compartir y colaborar en código. Características: Control de versiones con Git, gestión de problemas (issues), y revisión de código mediante pull requests.

    3. Inspección de código desde el navegador.

      4. Descripción: Herramientas de desarrollo integradas en navegadores web. Uso: Permiten a los desarrolladores inspeccionar y modificar HTML, CSS y JavaScript en tiempo real. Características: Acceso a la consola de JavaScript, depuración, análisis de rendimiento y visualización de la estructura del DOM.

Tema 2: El Diseño De Una Página Web

  1. tipos de paginas wed

    2. Páginas Estáticas: Contenido fijo, requiere cambios manuales. Páginas Dinámicas: Contenido que cambia según la interacción del usuario. Páginas de Comercio Electrónico: Para comprar y vender productos en línea. Páginas de Portafolio: Muestran el trabajo y habilidades de un individuo o empresa. Blogs: Publicaciones en formato de artículos, generalmente cronológicos. Páginas de Redes Sociales: Facilitan la interacción entre usuarios. Páginas de Noticias: Informan sobre eventos actuales. Páginas de Foros: Espacios para discusión e intercambio de ideas.

  2. puntos basicos en la elaboracion de una pagina wed.

    3. Planificación: Definir objetivos y audiencia. Diseño: Crear wireframes y elegir un estilo visual. Desarrollo: Usar HTML, CSS y JavaScript. Contenido: Redactar textos y optimizar para SEO. Pruebas: Revisar funcionalidades y compatibilidad. Lanzamiento: Publicar en un servidor y promocionar. Mantenimiento: Actualizar contenido y monitorear rendimiento.

Tema 3: Estructura De Una Página Web

  1. Jerarquía de directorio de un proyecto web.

    2. La jerarquía de directorio en un proyecto web organiza los archivos y recursos de manera que sea fácil de mantener y entender. Incluye carpetas para: CSS: Archivos de estilos que definen la apariencia. JavaScript: Archivos de scripts que añaden interactividad. Imágenes: Recursos gráficos utilizados en la página. Fuentes: Tipografías personalizadas. Páginas HTML: Documentos individuales que componen el sitio. Documentación: Información sobre el proyecto, como guías o notas.

  2. Elementos de estructura para el diseño de una página Web: Concepto

    3. Los elementos fundamentales para diseñar una página web son: Encabezado: La parte superior de la página que suele incluir el logo y el menú de navegación. Cuerpo: La sección principal donde se presenta el contenido, incluyendo textos, imágenes y otros elementos. Barra lateral: Espacios que pueden contener enlaces adicionales, menús o publicidad. Pie de página: La parte inferior que suele incluir información sobre derechos de autor, enlaces a políticas y contacto.

    1. Regla utilizada para validar y certificar una página web

      2. Las páginas web deben cumplir con estándares establecidos por el W3C. La validación se realiza a través de herramientas que revisan el código para asegurar que no contenga errores y que esté alineado con las mejores prácticas del desarrollo web.

    2. Reglas para escribir HTML.

      3. Las reglas para escribir HTML son fundamentales para asegurar que el documento sea accesible y funcione correctamente. Estas reglas incluyen: La correcta utilización de las etiquetas de HTML. El uso de atributos de manera clara y coherente. Mantener una estructura semántica que mejore la comprensión del contenido.

    3. Estructura básica de un archivo web

      4. La estructura de un archivo HTML incluye secciones que organizan el contenido de la página, permitiendo a los navegadores interpretar y mostrar la información correctamente. Cada sección tiene un propósito específico, desde definir el tipo de documento hasta incluir contenido visual y textual.
